# PRD 三轮 Review

## Review 1：商业化与产品定位

结论：通过，但需要把“试点首厂 -> 扩厂复制”的路径直接体现在产品里。

- 已满足：明确 ICP、采购委员会、套餐、收费维度、实施费、模块费、AI 用量、集成费。
- 已满足：没有把产品做成单纯 MES demo，而是定位为 ERP 与车间之间的制造运营平台。
- 已满足：ROI 指标可以支持 ToB 销售，包含准交率、库存、质量、上线周期、接口成功率。
- 风险：当前应用 UI 仍更像内部工作台，缺少客户成功/实施蓝图入口。
- 行动：新增“实施蓝图”模块，展示上线阶段、Owner、验收指标和商业套餐边界，并支持阶段推进、顺序约束、事件流与审计记录。

## Review 2：不同 ToB 用户是否拿起来就能用

结论：部分通过，必须增加角色化入口。

- 已满足：PRD 覆盖 COO、计划、生产、质量、仓储、采购、设备、财务、IT、供应商。
- 已满足：每类角色有首次登录、每日动作、成功指标。
- 缺口：当前应用默认是总览驾驶舱，用户不知道“我是谁，我今天该做什么”。
- 行动：新增全局角色切换器和角色化任务启动台。用户选择角色后，首页展示该角色的今日目标、关键动作、跳转模块和业务解释。

## Review 3：是否符合黑湖对标方向

结论：方向通过，仍需补齐“云端配置 + 专业实施 + 多角色协同”的产品表达。

- 对齐点：覆盖订单、排程、物料、生产、质检、设备、工厂建模、数据智能、流程规则、开放接口。
- 对齐点：强调快速上线、按需配置、多工厂协同、低维护 SaaS、审计与接口。
- 对齐点：AI 智能体被设计为业务动作建议，而不是装饰性聊天。
- 差距：黑湖强调实施顾问、角色权限组件化、浏览器即用、移动端/IoT；当前应用移动/PDA 与低代码表单还在路线图。
- 行动：第一轮先做角色化工作台和实施蓝图；后续迭代低代码配置、移动/PDA、供应商门户、行业模板。

## 本轮迭代验收标准

- PRD 文件存在，覆盖商业化、用户、模块、实施、路线图、指标。
- 应用中能切换不同 B 端角色，首页展示该角色“拿起来怎么用”。
- 侧边栏新增实施蓝图，表达首厂上线与扩厂复制流程。
- 实施蓝图阶段来自后端快照，可点击完成当前阶段，后续阶段必须等待前序完成。
- 阶段推进后必须写入实时事件与审计日志。
- UI 不再只像 demo 看板，而是体现销售、实施、使用、运营的完整 SaaS 逻辑。
